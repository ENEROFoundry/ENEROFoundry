# Principles - Overview

OEO Foundry follows a shared set of principles to ensure that ontologies are logically well-formed, scientifically accurate, and interoperable.
Our commitment to FAIR and open science drives the development of reusable knowledge structures for the energy domain.

- **1. Open** - The ontology must be freely accessible and usable by all under a suitable open license.
- **2. Common Format** - The ontology is made available in a common formal language in an accepted concrete syntax.

## Specification
- The ontology must be provided as an OWL file
- In addition, other file formats can be provided

## Recommendation
- The filename should be the ontology prefix: `oeo.owl`


# 3. Upper Ontology Structure

## Definition
The ontology must be built upon the Basic Formal Ontology (BFO).

## Specification
- The ontology must be based on BFO


# 4. Identifier Space

## Definition
The ontology must have a unique name, a distinct prefix, and a permanent identifier.

## Specification
- The ontology name must reflect its scope
- The prefix should be concise and derived from the ontology name


# 5. Versioning

## Definition
The ontology is under version control and all versions are released and stored.

## Specification
- The ontology files are stored in a git repository
- The releases follow semantic versioning (Major.Minor.Patch)


# 6. Scope

## Definition
The ontology must have a clearly specified scope and content that adheres to that scope.

## Specification
- The scope of an ontology is the extent of the domain or subject it intends to cover
- Ideally the scope should be fairly narrow


# 7. Naming Conventions

## Definition
The names (primary labels for elements) (classes, properties, etc.) in an ontology must be intelligible to scientists and amenable to natural language processing. Primary labels should be unique among  OEO Library ontologies.



# 8. Textual Definitions

## Definition
The ontology has textual definitions for the majority of its classes and for top level terms in particular.

## Specification


# 9. Documentation

## Definition
The owners of the ontology should strive to provide as much documentation as possible.


# 10. Commitment To Collaboration

## Definition
 OEO Foundry ontology development, in common with many other standards-oriented scientific activities, should be carried out in a collaborative fashion.
